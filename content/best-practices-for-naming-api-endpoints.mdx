# Best Practices For Naming API Endpoints

<TimeStamp updated="February 27, 2022, 9:20  PM ET"/>

* Use nouns, not verbs.
* Use plural names if they point to a collection of items.
* Use forward slashes to indicate hierarchy
* Use a comma to access more than one resource
* Use query parameters to sort or filter collections
* Use only lowercase letters
* Use dashes (-) instead of underscores (\_)
* Use American English
* Use intuitive names.
	* Names should read like prose
	* Use the simplest and most common word for a concept
* Use full words instead of abbreviations
* Don't include file extentions
* Don't use a trailing forward slash
* Name consistently
